# 💪 AI 一对一健身助手 (Fitness Assistant)

基于大语言模型的私人定制健身方案生成系统。通过 Streamlit 构建的现代化 Web 界面，为用户提供从身体素质评估到深度健身/饮食计划的一站式 AI 指导。

## 🌟 核心特性

-   **🎯 深度私人定制**：基于 13 项核心维度（身体数据、伤病史、训练环境、饮食作息等）生成极具针对性的健身方案。
-   **👤 多用户管理系统**：
    -   支持在侧边栏快速切换历史客户、新增客户。
    -   **数据隔离**：每个客户的数据独立存储在 `data/users/<姓名>/` 文件夹下。
    -   **秒速查看**：已有方案的用户可直接点击“查看方案”，无需重复生成。
-   **🧠 智能合并更新 (KISS 原则)**：为老客户更新方案时，AI 会读取旧报告并结合新需求进行逻辑合并，输出一份全新的精简文档，彻底避免内容无限堆叠。
-   **🏃 沉浸式加载体验**：
    -   **异步处理**：采用多线程技术，AI 在后台计算时前端界面保持流畅。
    -   **进度模拟**：内置 3 分钟平滑进度条（0-90%）及 10 条精选健身知识轮播，缓解等待焦虑。
-   **📋 专业内容输出**：
    -   **评估报告**：涵盖 BMI 分析、风险评估、灵活性预估等深度维度。
    -   **健身计划**：细化至动作、组数、次数、RPE 强度、组间休息。
    -   **营养指导**：基于 TDEE 计算的宏量营养素（碳水/蛋白/脂肪）克数建议及三餐示例。
-   **🎨 现代圆润 UI**：通过自定义 CSS 注入，实现输入框聚焦光晕、圆角容器及优雅的侧边栏布局。

## 🛠️ 技术栈

-   **Frontend/Backend**: [Streamlit](https://streamlit.io/) (Python 纯代码全栈实现)
-   **AI Engine**: Qwen 3.5 27B (通过 OpenAI 兼容接口调用)
-   **Data Storage**: 本地 Markdown 文件持久化
-   **Concurrency**: Python `threading` 多线程异步处理

## 🚀 快速开始

### 1. 环境准备
确保您的系统中已安装 Python 3.8+。

### 2. 安装依赖
```powershell
pip install -r requirements.txt
```

### 3. 配置文件
在项目根目录下创建 `.env` 文件（参考 `.env.template`）：
```ini
API_KEY=您的API密钥
BASE_URL=您的模型接口地址
MODEL_ID=qwen3.5-27b
```

### 4. 运行应用
```powershell
streamlit run app.py
```

## 🔒 安全与隐私

-   **环境变量**：所有 API 凭证均通过 `.env` 管理，且已配置 `.gitignore` 严防泄露。
-   **数据本地化**：所有生成的评估报告和计划均存储在本地 `data/` 目录下，不上传云端，保护客户隐私。

## 📝 使用建议

1.  **初次使用**：在左侧边栏点击“新建客户”，在主页面填写详细信息。
2.  **更新方案**：在侧边栏选中已有客户，点击“修改信息”，提交新变动后 AI 将自动更新旧文档。
3.  **专业度**：建议如实填写“伤病情况”，AI 会据此自动排除高风险动作并给出康复建议。
